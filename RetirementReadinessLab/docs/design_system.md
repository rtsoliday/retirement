# Design System

## Design Intent

Retirement Readiness Lab should feel calm, premium, clear, and serious. The interface should help people scan decisions quickly rather than read dense financial prose.

## Palette

| Token | Color | Use |
| --- | --- | --- |
| Background | `#F7F8F5` | App background |
| Surface | `#FFFFFF` | Cards and sheets |
| Primary | `#176B5B` | Main actions, selected nav |
| Primary Dark | `#0E463C` | Strong emphasis |
| Accent | `#D6A23A` | Comparison highlights |
| Success | `#2F7D4E` | Healthy result states |
| Caution | `#B7791F` | Watch states |
| Risk | `#B94A48` | Poor result states |
| Text | `#17201C` | Primary text |
| Muted Text | `#65706A` | Secondary text |
| Divider | `#DDE3DE` | Borders and separators |

## Component Rules

- Use cards for discrete repeated objects and dashboard metrics.
- Keep card corner radius at 8dp.
- Use icons in navigation and command buttons.
- Use segmented choices for option sets.
- Use switches for binary strategy toggles.
- Use sliders for ages and sensitivity sweeps.
- Use compact chart cards for mobile and full-screen charts for detail.

## Dashboard Visual Hierarchy

1. Readiness probability.
2. Short outcome sentence.
3. Portfolio percentile chart.
4. Key metrics.
5. Biggest risk and next scenario.

## Chart Rules

- Use green for successful/healthy ranges.
- Use amber for caution.
- Use red only for serious risk.
- Do not rely on color alone.
- Label dollars in rounded values.
- Avoid dense scatter plots on the main dashboard.

## Tone

Use direct, measured language:

- "82% of simulations lasted through age 95."
- "Healthcare costs are the largest modeled pressure."
- "Try delaying Social Security or reducing spending by 5%."

Avoid:

- "You are safe."
- "Guaranteed retirement."
- Fear-driven copy.
